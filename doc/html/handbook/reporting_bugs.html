Chapter 11. Reporting bugs

While the author of UltraDefrag Driver has taken great pains to ensure that UltraDefrag operates properly, no software is perfect. As such bugs are inevitable. The author accepts bug reports and will attempt to fix issues that are brought to his attention. In order to facilitate this the author has built logging mechanisms into UltraDefrag. This document describes how to get these logs and attach them to your bug report.

Also if you have any problems with scheduler, documentation, web site appearance and any other parts of the UltraDefrag project, please send a bug report with the following data attached:

  1. Screenshot (required for GUI related bugs).
  2. %windir%\UltraDefrag\logs directory contents. If this directory is empty please start DbgView program before UltraDefrag runs to see what happens. Also DbgPrint Logger can be used to collect debugging information. Since the 2.1.0 version of the Ultra Defragmenter these tools are optional and should be used only if debugging logs are empty due to some logging related bugs.
  3. If you have seen Blue Screen Of Death (BSOD), include in your report numbers shown on BSOD and Mini Crash Dump file (required!) that can be found in %windir%\Minidump directory.

Please report all problems and bugs via the bug tracker. Don't forget to register on sourceforge.net before. As an alternative you can send your report directly to the authors by email.
